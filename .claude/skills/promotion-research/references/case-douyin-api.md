## 三、用户画像

### 目标人群特征

- **性别**：男性 80%，TGI 155
- **年龄**：31–40 岁占比最高，24–30 岁 TGI 最高
- **地域**：广东（14.82%，TGI 136）、浙江（8.01%）、江苏（7.77%）
- **特征**：有一定技术背景但非专业开发者，知道 Claude 好用但不清楚怎么买

### 散户真实路径

1. 看到抖音/朋友圈说"OpenClaw 好厉害"
2. 想试试，搜"怎么用"
3. 发现要注册 + 翻墙 + 虚拟信用卡
4. **在这里放弃，或求助他人** ← 真正目标用户

### 用户分层

| 类型 | 占比 | 日均消息 | token焦虑 | 付费意愿 |
|------|------|---------|---------|---------|
| 轻度用户 | 68% | 5–15条 | 几乎感受不到 | 低 |
| 中度用户 | 23% | 20–35条 | 偶尔卡 quota | 中 |
| 重度用户 | 9% | 40+条 | **焦虑最严重** | **高** |

**结论**：真正有付费意愿的是那 9%，但他们是开发者会自己找方案。目标用户是**看了视频、想用但看不懂教程的普通人**。

### 白嫖文化（技术圈已饱和）

- AnyRouter 公益站：注册送 $100 + 邀请送 $50
- LinuxDo 公益 API 站：免费但不稳定，"3个月倒闭一批"
- 知乎薅 token 攻略已很成熟

**结论**：技术用户几乎不需要付钱，**目标必须是散户**。

## 四、合作方案分析

### 对方情况（池子）

- 目前只做 toB 业务，想拓展 toC
- 有稳定商务关系和供给渠道
- 表示"不是通过共享的方式"（具体机制待确认）

### 商业模式

- **进货价**：0.5 元/unit
- **销售价**：4–5 元/unit（毛利 8–10 倍）
- **目标客群**：散户、toC、非开发者

### 风险矩阵

| 风险类型 | 等级 | 分析 |
|---------|------|------|
| 合规风险 | ⚠️ 中 | toB 有合同/发票较正规，但 Anthropic 无官方"分销商"机制 |
| 封号风险 | ⚠️ 中（近期低） | 对方称"稳稳的"，但 Anthropic 风控持续升级 |
| 供给稳定性 | ✅ 较高 | 有 Max 额度支撑，但上限决定业务规模 |
| 热度衰减风险 | ⚠️ 高 | OpenClaw 可能昙花一现，需快速行动 |

### 分利方案建议

| 方案 | 描述 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| **方案一（推荐）** | 纯佣金 10-15% | 轻资产，不垫资 | 主动权在对方 | 先试水 |
| 方案二 | 批发差价（如 1.5 元/unit） | 有定价权 | 需垫资压货 | 有资金实力 |
| 方案三 | 利润分成 40/60 | 深度绑定 | 需建结算机制 | 长期合作 |

**务实建议**：担忧是昙花一现，分利方案偏向"低承诺、快结算"——按月结算佣金，窗口期过了合作关系自然结束。

## 六、结论与建议

### 核心结论

1. **市场机会真实存在**：token/openclaw 搜索量巨大，内容供给稀缺
2. **窗口期有限**：2-4周内行动，否则竞争者涌入
3. **目标用户明确**：非技术散户，卡在"想用但不会买"环节
4. **合作可行但谨慎**：建议"低承诺、快结算"模式试水
