# 问题拆解（Overcome Any Problem）

用结构化思维将障碍转化为可执行的步骤。结构化思考胜过蛮力。

## 触发方式

**手动触发：**
- "帮我解决这个问题"
- "卡在XXX上了"
- "怎么解决XXX"
- "拆解一下这个问题"
- "克服这个困难"

## 核心流程

```
Define（定义）→ Analyze（分析）→ Generate Options（生成选项）→ Choose（选择）→ Execute（执行）→ Review（回顾）
```

## 8个分析角度

| 角度 | 问题 |
|------|------|
| **功能** | 需要实现什么？ |
| **技术** | 技术上可行吗？ |
| **时间** | 时间约束/期限是什么？ |
| **资源** | 有哪些可用资源？ |
| **风险** | 潜在风险是什么？ |
| **利益相关者** | 谁受影响？ |
| **先例** | 类似问题如何解决？ |
| **创意** | 有非常规思路吗？ |

## 执行步骤

### Step 1: 定义问题

**明确问题陈述：**
- 什么在阻碍你？
- 成功是什么样子的？
- 约束条件是什么？

**输出格式：**
```
🎯 问题定义
- 当前状态：[描述现状]
- 目标状态：[描述目标]
- 核心障碍：[主要阻碍]
- 约束条件：[时间/资源/其他限制]
```

### Step 2: 分析问题

从8个角度分析，找到最紧的约束。

**关键原则：**
- 从约束开始 —— 解决最紧的约束
- 从原语思考 —— 最小构建块
- 并行优于串行 —— 找到独立的工作流

### Step 3: 生成选项

头脑风暴 3+ 个解决方案，先不过滤。

```
💡 方案A：[描述]
💡 方案B：[描述]
💡 方案C：[描述]
```

### Step 4: 选择方案

选择最高杠杆的路径，说明理由。

```
✅ 选择方案：[X]
理由：[为什么选这个而不是其他]
```

### Step 5: 行动计划

**拆解为可执行步骤：**

| 步骤 | 任务 | 依赖 | 预计时间 | 状态 |
|------|------|------|----------|------|
| 1 | [任务描述] | - | [时间] | 待办 |
| 2 | [任务描述] | 步骤1 | [时间] | 待办 |
| 3 | [任务描述] | 步骤1 | [时间] | 待办 |

**标记：**
- 🔴 关键路径（必须完成）
- 🟢 可并行（不依赖其他步骤）
- 🟡 有风险（可能出问题）

### Step 6: 进度跟踪

执行时更新状态，记录阻碍。

```
📊 进度更新
- 已完成：步骤1、步骤2
- 卡在：步骤3 - [具体问题]
- 调整：[应对措施]
```

### Step 7: 回顾

```
🔄 回顾
- 什么有效？
- 什么可以改进？
- 需要更新框架吗？
```

## 输出示例

```
🎯 问题定义
- 当前状态：record skill 没有自动触发，需要手动调用
- 目标状态：关键节点出现时自动记录
- 核心障碍：关键词匹配逻辑不准确
- 约束条件：不能影响系统性能

📊 分析
- 技术：触发机制依赖关键词识别，当前匹配规则太严格
- 时间：每次修复需要迭代测试
- 资源：需要分析历史对话数据

💡 生成选项
方案A：扩大关键词列表，覆盖更多表达方式
方案B：改为意图识别（LLM判断而非关键词匹配）
方案C：混合模式 —— 关键词 + 意图识别兜底

✅ 选择方案B
理由：意图识别更灵活，覆盖面更广，长期维护成本更低

📋 行动计划
| 步骤 | 任务 | 依赖 | 时间 |
|------|------|------|------|
| 🔴 1 | 设计意图识别提示词 | - | 30min |
| 🟢 2 | 收集测试用例（50+对话） | - | 20min |
| 🔴 3 | 实现意图识别 | 步骤1 | 60min |
| 🟡 4 | 集成到 record skill | 步骤3 | 30min |
| 🟢 5 | 测试验证 | 步骤4 | 30min |
```

## 关键原则

1. **从约束开始** - 解决最紧的约束
2. **从原语思考** - 最小构建块
3. **乘法而非加法** - 每个系统应该放大其下的一切
4. **并行优于串行** - 找到独立的工作流，同时执行
5. **结构化胜过蛮力** - 分解比硬冲更有效

## 使用场景

- 遇到复杂问题时
- 多次尝试未果时
- 需要制定行动计划时
- 项目规划/任务分解时

## 注意事项

- 问题定义要清晰，模糊问题无法解决
- 生成选项时不过滤，多角度思考
- 选择方案时说明理由，便于后续回顾
- 执行时及时更新进度，遇到阻碍及时调整
