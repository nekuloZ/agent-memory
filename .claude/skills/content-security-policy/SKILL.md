# Content Security Policy Skill

**Name**: `content-security-policy`  
**Description**: 内容安全策略 + 敏感信息脱敏 + 权限管理 + 敏感操作确认  
**Version**: v1.0  
**Author**: nekulo  

---

## 🎯 核心功能

本 Skill 为 OpenClaw 提供全方位的安全防护：

1. **敏感信息自动脱敏** - 拦截并脱敏 IP、API Key、Token、路径等
2. **权限分级管理** - L0(最高权限) / L1(授权用户) / L2(普通) / L3(群聊)
3. **敏感操作确认** - 删除文件、修改多维表格等操作需二次确认
4. **安全事件日志** - 记录所有敏感操作到本地文件

## 📝 系统安全指令

当处理用户请求时，必须遵循以下安全规则：

### 1. 敏感信息脱敏

在**任何输出**（包括回复、日志、文档）中，必须脱敏：

| 类型 | 检测模式 | 脱敏后 |
|------|---------|--------|
| IPv4 | `xxx.xxx.xxx.xxx` | `x.x.x.x` |
| IPv6 | `[IPv6格式]` | `[IPv6地址]` |
| API Key | `sk-xxx`, `AKLTxxx`, `wx...` | `...[密钥]` |
| Access Token | `eyJ...` JWT格式 | `[JWT Token]` |
| 服务器路径 | `/root/...`, `C:\Users\Admin\...` | `/path/to/...` |

**规则**: 
- 在回复用户前，扫描内容中的敏感信息
- 发现敏感信息 → 立即脱敏 → 再输出
- 绝不输出完整的真实 IP、密钥、Token

### 2. 权限判断流程

```
收到请求
    ↓
[1] 识别用户身份
    ├─ OpenID = ou_18a1d22fd6644c85e71126a0fb870645 → L0 (最高权限)
    ├─ 在授权列表中 → L1 (授权用户)
    ├─ 私聊普通用户 → L2
    └─ 群聊成员 → L3
    ↓
[2] 判断操作类型
    ├─ 敏感操作 (删除/修改结构/群发) → 需要确认
    ├─ 普通操作 → 直接执行
    └─ 只读操作 → 直接执行
    ↓
[3] 权限检查
    ├─ L0 → 跳过确认，直接执行
    ├─ L1-L3 + 敏感操作 → 必须向用户确认
    └─ L1-L3 + 普通操作 → 直接执行
```

### 3. 敏感操作确认话术

当非 L0 用户请求敏感操作时：

> ⚠️ **敏感操作确认**
> 
> 您请求执行以下敏感操作：
> - 操作类型: `[删除文件/修改表格结构/群发消息]`
> - 目标对象: `[对象名称]`
> 
> 请确认是否继续？
> - 回复 "确认" 继续执行
> - 回复 "取消" 中止操作

**L0 用户**执行敏感操作时，记录日志但无需确认。

### 4. 危险指令拦截

以下指令**绝对禁止**：

| 类别 | 指令示例 | 处理方式 |
|------|---------|---------|
| 远程连接 | `ssh`, `scp`, `telnet`, `rdp` | 直接拒绝 |
| 系统删除 | `rm -rf /`, `rm -rf ~`, `del /f /s /q C:\` | 直接拒绝 |
| 敏感读取 | `cat ~/.ssh/id_rsa`, `cat .env` | 脱敏后输出 |

**拒绝话术**:
> 🚫 **操作被拒绝**
> 
> 出于安全考虑，我无法执行 `[SSH连接/系统删除/敏感文件读取]` 操作。
> 如有服务器操作需求，请联系管理员手动执行。

## 📊 安全事件日志

### 日志格式

所有敏感操作和权限相关事件必须记录：

```json
{
  "timestamp": "2026-03-16T14:00:00+08:00",
  "event_type": "sensitive_operation|permission_denied|data_desensitized",
  "user_id": "ou_xxx",
  "user_level": "L0|L1|L2|L3",
  "operation": "tool_name.action",
  "target": "操作对象",
  "decision": "allowed|denied|confirmed",
  "reason": "决策原因"
}
```

### 日志文件位置

```
~/.openclaw/security-logs/
├── security-events-2026-03-16.log
├── security-events-2026-03-17.log
└── archive/
    └── security-events-2026-02.log.gz
```
